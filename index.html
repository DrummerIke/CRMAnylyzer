Операционный центр отбора клиентской базы

Сначала загружаем базу. Потом выбираем только те сегменты, которые реально есть в этом файле. Далее задаём гибкие условия по любым столбцам, определяем приоритет отбора, смотрим по каждому сотруднику, сколько найдено, вручную ставим лимиты и в конце выгружаем только номера для запуска коммуникаций.

1

Загрузка базы и поля

Файл загружается один раз, дальше система работает уже по этой базе.

Файл не загружен
Поля будут определены автоматически
Дата в превью показывается как DD.MM.YYYY, суммы — с разделителями и знаком рубля.
2

Сегменты из этой выгрузки

Показываются только значения сегментов, которые реально есть в файле.

Загрузите файл, затем здесь появятся сегменты.
Сегменты сгруппированы красиво: блок клиента → статус. Остальные не показываются.
3

Гибкие условия отбора

Любой столбец, понятный оператор, отдельные поля для дат и чисел.

Примеры: дата последнего контакта до 31.01.2025, оборот больше 100000, сегмент содержит VIC.
4

Приоритет отбора внутри сотрудника

Именно тут решается, какие записи попадут в первые 50, 80 или 100.

Порядок критериев важен: первый решает больше всего. Например: сначала старые контакты, потом больший оборот.
5

Сотрудники и квоты

Сначала система показывает, сколько нашлось по каждому сотруднику. Потом ты задаёшь, сколько взять.

После нажатия «Посчитать отбор» появятся сотрудники и найденные по ним записи.
Если нужно исключить сотрудника — поставь галочку. Если нужно уменьшить, просто измени лимит.
