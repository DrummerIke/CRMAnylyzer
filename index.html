Операционный центр отбора клиентской базы

1) загружаем файл, 2) смотрим только реальные сегменты из этой выгрузки, 3) задаём гибкие условия по любым полям, 4) выбираем приоритет отбора, 5) видим нагрузку по сотрудникам, 6) ограничиваем количество вручную, 7) выгружаем только номера.

2

Сегменты из этой выгрузки

Показываются только значения, которые есть в файле. Сначала VIC, потом русские, потом прочие.

Загрузите файл, затем здесь появятся сегменты.
Сегменты отображаются полными названиями, без сокращений и без лишних значений.
3

Гибкие условия отбора

Типы полей определяются по смыслу: дата / число / цифры / текст.

Телефон и Номер карты — цифры. Общ.сумма, Сумма покупок за период, Из них сумма Прямых продаж, Кол-во штук товаров — числа. Посл.комм, Посл.покупка, ДР, Дата коммуникации — даты.
4

Приоритет отбора

Оставлены только поля, которые ты перечислил. Для категории и пола — свои понятные направления.

Категория сортируется по логике: VIC → Искра → Семья → Друзья → Знакомые → Новички. Пол — М → Ж или Ж → М.
