Операционный центр отбора клиентской базы

1) загружаем базу, 2) выбираем сегменты только из этой выгрузки, 3) задаём гибкие условия по любым полям, 4) настраиваем приоритет отбора, 5) смотрим по каждому сотруднику, сколько нашлось, 6) задаём квоты и потолок, 7) выгружаем только номера для запуска коммуникаций.

2

Сегменты из этой выгрузки

Показываются только значения, которые реально есть в базе. Полные названия, без сокращений.

Загрузите файл, затем здесь появятся сегменты.
Сначала идут сегменты VIC, потом остальные по алфавиту.
3

Гибкие условия отбора

Любой столбец, понятные операторы, отдельные поля для дат и чисел.

Примеры: дата последнего контакта до 31.01.2025, оборот больше 100000, сегмент содержит VIC.
4

Приоритет отбора внутри сотрудника

Именно тут решается, какие записи попадут в первые 50, 80 или 100.

Порядок критериев важен: первый решает больше всего. Например, сначала старые контакты, потом больший оборот.
5

Сотрудники и квоты

Верхний предел задаётся один раз и работает для всех сотрудников.

Можно уменьшать вручную или исключать сотрудника галочкой.
